Recipient.Notify Property

Sets the kind of notification (report) desired for this recipient.

public Notify Notify {get; set;}

Property Value

A notification can be requested for successful delivery, failed delivery, and delayed delivery. These can be combined by using bit-wise combinations. For example, 

Recipient r = new Recipient("me@here.com");
r.Notify = Notify.Failure | Notify.Delay;
asks for notification on falied delivery and on delayed delivery.

Remarks

By default, notification is only requested upon delivery failure. Note that while PowerMTA does not support notification for delivery delays, the corresponding DSN flag is passed on to the receiving mailer, which may support it.
